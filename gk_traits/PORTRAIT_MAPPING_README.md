# Portrait Mapping CSV

`PORTRAIT_MAPPING.csv` lists every species portrait used by the gk_traits
category model, with a `=IMAGE()` formula that renders the portrait from
the Paradox wiki inline.

## How to use in Google Sheets

1. New Sheet, then **File > Import > Upload > PORTRAIT_MAPPING.csv**.
2. On the import dialog, set **Convert text to numbers, dates, and formulas: Yes**.
   (Without this, the IMAGE() formulas land as literal strings and no images render.)
3. Widen the **Preview** column and set row height to ~80 px so portraits are visible.
4. Optional: freeze the first row and the first two columns.

## Workflow

For each row, decide its two categories and edit columns D (Category A) and E (Category B).
Mark status in column F:

- `draft` = auto-derived default, not yet reviewed by you
- `done`  = you've confirmed the two categories are correct
- `?`     = filename uncertain (image won't render) OR you want a second look

Status defaults: most rows ship as `draft`. Rows with `?` are ones where I
couldn't verify the wiki filename - those will show broken images in Sheets
until you fix the filename in column B.

## Regenerating

The CSV is generated by `tools/build_portrait_csv.py`. If you correct a
wiki filename in the script, regenerate with:

    python3 tools/build_portrait_csv.py > PORTRAIT_MAPPING.csv

Note that regenerating overwrites any edits to Category A/B/Status/Notes in
the CSV. Once you start editing in Sheets, treat the Sheet as the source of
truth - the script is only for the initial scaffold.

## Known uncertain rows (status = ?)

7 rows ship as uncertain - the wiki name is a guess and may 404:

- `art2`, `art4`  - wiki API didn't enumerate Arthropoid_slender_02/04
- `stargazer_01`  - Stargazer DLC, no wiki page found
- `pla16_baol`    - Ancient Relics Baol plantoid
- `lith_human_inf`, `lith2_inf` - infernal+lithoid hybrid portraits
- `default_robot` - the unstyled fallback robot portrait

If you find the right wiki filenames, edit `tools/build_portrait_csv.py`
and regenerate, or just edit the CSV/Sheet directly.

## Trait gate derivation

When the time comes to write trait files, the gates derive from this table:
for each category X, the trait's `portrait_override` is the union of all
portrait IDs where Category A or Category B equals X. The script that does
that derivation isn't written yet - it'll consume `PORTRAIT_MAPPING.csv`
as input.
