# gk_traits: extend the empire-designer UI categories with our portrait_sets so
# cross-tagged and merged portraits appear in the right dropdowns.
#
# Stellaris MERGES the inner `sets` list across files when the same category
# block appears more than once - vanilla's sets are preserved automatically.
# We only list our NEW gk_* sets here; redeclaring vanilla sets would cause
# duplicates.
#
# A set may be listed under MORE THAN ONE category: that is how a portrait shows
# in several dropdowns while keeping one species_class. gk_gauntlet_cybernetic_reptilians
# is listed under both `reptilians` and `cybernetics`, mirroring vanilla's
# cybernetic_avians (species_class = AVI, listed under `avians` and `cybernetics`).
#
# DLC gating lives inside each gk_* portrait_set's conditional_portraits block
# (see 01_gk_crosslist_sets.txt) - this file is strictly category membership.
